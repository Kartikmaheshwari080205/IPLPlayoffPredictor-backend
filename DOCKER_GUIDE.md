# Docker Setup Guide for IPL Playoff Predictor Backend

## ⚡ Quick Start (Recommended)

### Windows Users
```bash
cd IPLPlayoffPredictor-backend
docker-setup.bat
```

### Mac/Linux Users
```bash
cd IPLPlayoffPredictor-backend
bash docker-setup.sh
```

This automatically builds the image and starts the container!

---

## 📝 Manual Setup (If Preferred)

### Build the Docker Image

```bash
cd IPLPlayoffPredictor-backend
docker build -t ipl-predictor:latest .
```

### Run with Docker Compose (Recommended)

```bash
# Start the container
docker-compose up -d

# View logs
docker-compose logs -f

# Stop the container
docker-compose down
```

### Or Run with Docker Directly

```bash
docker run -d \
  -p 8000:8000 \
  -v $(pwd)/matches.txt:/app/matches.txt \
  -v $(pwd)/h2h.txt:/app/h2h.txt \
  -v $(pwd)/ipl_json:/app/ipl_json \
  --name ipl-api \
  ipl-predictor:latest
```

---

## 🗂️ Data Persistence

The Docker container mounts these files as volumes so they persist and can be updated:

- **`matches.txt`** - Updated by `nightly_job.py` and `refresh_ipl_data.py`
- **`h2h.txt`** - Head-to-head data (can be updated externally)
- **`ipl_json/`** - Match data directory (updated by refresh scripts)
- **`probabilities.txt`** - Generated by predictor binary (read-only in API)

### How It Works:
1. Container reads files from `/app/` inside the container
2. These are mapped to your local files via volumes
3. When `nightly_job.py` updates `matches.txt`, it updates the **local file**
4. Next time the API runs, it reads the updated file

## 🔄 Running the Nightly Job in Docker

You can run the nightly job inside the Docker container:

```bash
# Run nightly job once
docker-compose exec backend python nightly_job.py

# Or run predictor directly
docker-compose exec backend ./predictor
```

Or schedule it with cron (Linux/Mac):
```bash
0 2 * * * cd /path/to/IPLPlayoffPredictor-backend && docker-compose exec backend python nightly_job.py
```

## 🐛 Troubleshooting

### Container won't start
```bash
docker-compose logs
# Check for errors, usually permission issues
```

### Files not updating
- Make sure volumes are mounted correctly
- Check: `docker inspect <container-id>` for Mounts section

### Port already in use
- Change port in `docker-compose.yml`: `"8001:8000"` instead of `"8000:8000"`

### Permission denied on files
```bash
# Linux/Mac: Make scripts executable
chmod +x docker-setup.sh

# Windows: Run Command Prompt as Administrator
```
